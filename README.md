# reso_db

## Windows에서 git push 빠르게 따라하기

1. **Git 설치**
   - [Git for Windows](https://git-scm.com/download/win)을 설치하면 `Git Bash`와 `Git CMD`가 함께 제공됩니다. 별도의 추가 커맨드라인을 설치할 필요 없이, 설치 과정에서 추가되는 터미널을 바로 사용할 수 있습니다.
   - GUI를 선호하면 GitHub Desktop 같은 도구도 선택 가능하지만, 아래 단계는 기본 커맨드라인 기준입니다.

2. **저장소 클론 또는 최신 상태로 갱신**
   - 새로 작업을 시작할 때는 저장소를 클론합니다.
     ```bash
     git clone <repo-url>
     cd <repo-folder>
     ```
   - 기존에 클론한 경우 최신 변경을 받아옵니다.
     ```bash
     git pull
     ```

3. **작업 및 변경 사항 확인**
   - 파일을 수정하거나 추가/삭제한 뒤 현재 변경 상태를 확인합니다.
     ```bash
     git status
     ```

4. **커밋 준비**
   - 변경한 파일을 스테이징합니다.
     ```bash
     git add <파일명 또는 경로>
     ```

5. **커밋 만들기**
   - 의미 있는 메시지로 커밋을 생성합니다.
     ```bash
     git commit -m "짧고 명확한 변경 설명"
     ```

6. **원격 저장소로 push**
   - 현재 브랜치를 원격에 올립니다.
     ```bash
     git push
     ```
   - 처음 push하거나 새 브랜치를 사용할 때는 원격 브랜치를 지정해야 할 수 있습니다.
     ```bash
     git push -u origin <브랜치명>
     ```

7. **인증 안내**
   - push 시 자격 증명이 필요하면 **메시지 창(자격 증명 팝업)** 또는 터미널 내 입력 프롬프트가 나타납니다. Git for Windows를 기본 설정으로 설치했다면 추가 설치 없이 이 창을 통해 사용자명/토큰을 입력할 수 있습니다.
   - 토큰을 미리 캐시하려면 `git config --global credential.helper manager-core`(기본값) 설정을 사용하거나, 환경에 맞는 크리덴셜 매니저를 설정하면 됩니다.

8. **완료 확인**
   - push 후에는 원격 저장소(예: GitHub)의 브랜치/PR 화면에서 커밋이 반영됐는지 확인합니다.

> 요약: Git for Windows 설치만으로 필요한 커맨드라인이 모두 제공되며, push 과정에서 인증이 필요하면 자동으로 터미널 입력 또는 크리덴셜 팝업이 나타나므로 별도 메신저나 추가 툴을 요청받지 않습니다.
